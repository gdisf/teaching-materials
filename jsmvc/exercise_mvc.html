Exercise: MVC



In this exercise, you will take an app and split it into a Model and View.

Go through these steps, and do as many as you can in the time we have:

  • Download this file and save it into your project folder. Open it up in your browser and make sure it works. You should see a list of videos, and when you click on them, a modal pops up that plays the video.
  • Read through the code and try to understand what it's doing.

  • Modeling The Data

  • Create a VideoModel object that represents a video. Your model object should take in a JSON object and store title, author, and youtubeID properties. It should also calculate an embed and preview URL. (Hint: Look at BookModel in the slides)
  • Create a VideoCollection object that represents a collection of videos. It should have an import method that takes in JSON and turns it into an internal items array of VideoModel instances. It should also have a sort method that sorts the items by title. (Hint: Look at BookCollection in the slides - 90% of the code in the slides)
  • Modify the code in document.ready to create a VideoCollection and sort it.
  • Change the for loop to iterate through the items in the VideoCollection> and pass them to the addVideoToList function.
  • Try loading the page now and see if your code works. Are there any errors? Look at the console and work through them one by one. You may have to modify a few things in your addVideoToList function.

  • Creating the View

  • Create a VideoItem view. It should be very similar to addVideoToList, but instead of appending to the global DIV (#videos-list), it should return a DIV.
  • In the for loop in document.ready, change the code to create a new VideoItem object, and append the resulting DIV to the global #videos-list DIV there instead.
  • Try loading the page and see if your code works!
  • Bonus: Test the reusability of your code now by creating a new DIV in the HTML #videos2-list and see if you can create new item views to insert in that DIV.

Make sure that you use your browser developer tools to make debugging easier while working on this. Check for errors, and use console.log() to figure out how far your code makes it, and what the values of your variables are along the way.
