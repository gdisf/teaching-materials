Exercises

Exercises to help explain how Data Modeling works!

Modeling People - Exercise 2

In this exercise you'll figure out how to model a person.

  • Create a file, call it users.json
  • In that file, create a blank array, like this: [ ]
  • In that array, create several empty objects. Example:
[
    {
        property: "value",
        property2: ["array", "of", "strings"],
        attribute: 27,
        thing: [24, 36, "47", "mixed array!"]
        sub_objects: [{name: "example", of: "subobjects"}, {remember: "Objects can", go: "in arrays too."}]
    },
    {
        objects: "Don't all have to have the same properties."
    }
]
  • You'll be creating two users modeled after actual people.
  • We want for them to both have usernames, passwords (you can store them as plain text for this exercise.)
  • Other attributes are up to you.
  • We're going to model Oprah Winfrey and Eleanor Roosevelt
  • Use their wikipedia profiles (linked) to gain some info about them (like birthdate, current age) and other trivia.
  • Bonus: Model yourself, and a friend of yours. Use real email addresses.
